= Informationen zu Red Hat OpenShift Virtualisierung
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Red Hat® OpenShift Virtualization ermöglicht Unternehmen die nahtlose Migration, den Betrieb und die Verwaltung traditioneller virtueller Maschinen (VMs) neben modernen Workloads auf einer einheitlichen Hybrid-Cloud-Anwendungsplattform. OpenShift Virtualization ist als Operator in jedem OpenShift Abonnement enthalten. Dieses Add-on für die OpenShift Container Platform bietet eine robuste Lösung zur Integration von VMs in containerisierte Anwendungen. Es nutzt Kernel-based Virtual Machine (KVM), den bewährten Linux-Kernel-Hypervisor, der für seine hohe Leistung, Skalierbarkeit und Stabilität bekannt ist, um Virtualisierung der Enterprise-Klasse für virtuelle Maschinen als native Objekte innerhalb von OpenShift bereitzustellen.

image:redhat-openshift-044.png["OpenShift-Virtualisierung"]



[[key-features-of-red-hat-openshift-virtualization]]
== Hauptmerkmale der Red Hat OpenShift Virtualisierung

**Hybride Workloads:** Migration und Betrieb von VMs mit nativen Cloud-Vorteilen sowie Verwaltung dieser zusammen mit Containern und serverlosen Workloads mithilfe eines einheitlichen Toolsets.

**Schrittweise Modernisierung:** Eine Modernisierung im eigenen Tempo ist möglich, indem Container eingeführt, Microservices implementiert oder KI integriert werden, ohne bestehende VMs zu beeinträchtigen.

**Kubernetes-Integration:** Nutzt benutzerdefinierte Kubernetes-Ressourcen, um Virtualisierungsfunktionen zu Ihrem OpenShift-Cluster hinzuzufügen.



[[capabilities-of-red-hat-openshift-virtualization]]
== Funktionen von Red Hat OpenShift Virtualisierung

**VM-Verwaltung:** Erstellen und Verwalten von Linux- und Windows-VMs.

**Vereinheitlichte Workloads:** Pod- und VM-Workloads können gleichzeitig innerhalb eines Clusters ausgeführt werden.

**Konnektivität:** Zugriff auf VMs über verschiedene Konsolen und CLI-Tools.

**VM-Operationen:** Import, Klonen und Live-Migration von VMs zwischen Knoten.

**Ressourcenverwaltung:** Verwaltung von Netzwerkschnittstellen und Speicherfestplatten, die an VMs angeschlossen sind.



[[compatibility]]
== Kompatibilität

Während sich OpenShift Virtualization konzeptionell von VMware vSphere unterscheidet, bietet es eine moderne Plattform, die speziell auf Virtualisierungsadministratoren zugeschnitten ist und vertraute Funktionen wie folgende bietet: **Live Migration:** Entspricht vMotion in vSphere.

**Speicherverwaltung:** Speicherklassen in OpenShift stehen im Einklang mit Storage Policy Based Management (SPBM) in vSphere.

**Persistenter Speicher:** Persistent Volume Claims (PVC) und Persistent Volumes (PV) entsprechen Datastores in vSphere.
