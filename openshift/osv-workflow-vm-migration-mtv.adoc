= Migrieren Sie eine VM von VMware zu einem Red Hat OpenShift-Cluster
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
VMs können von VMware auf einen OpenShift Cluster mithilfe des OpenShift Virtualization Migration Toolkits migriert werden. Bei dieser Migration werden das Migration Toolkit for Virtualization (MTV) installiert, Quell- und Zielanbieter erstellt, ein Migrationsplan erstellt und eine Kalt- oder Warmmigration durchgeführt.

.Kalte Migration
[%collapsible%open]
====
Dies ist der Standardmigrationstyp. Die virtuellen Quellmaschinen werden während des Kopierens der Daten heruntergefahren.

====
.Warme Migration
[%collapsible%open]
====
Bei dieser Art der Migration werden die meisten Daten während der Vorkopierphase kopiert, während die virtuellen Quellmaschinen (VMs) ausgeführt werden.  Anschließend werden die VMs heruntergefahren und die verbleibenden Daten während der Umstellungsphase kopiert.

====

== Migration von VMs von VMware zu OpenShift Virtualization mit dem Migration Toolkit for Virtualization

In diesem Abschnitt wird beschrieben, wie das Migration Toolkit for Virtualization (MTV) verwendet wird, um virtuelle Maschinen von VMware auf OpenShift Virtualization zu migrieren, die auf der OpenShift Container Platform läuft und mit NetApp ONTAP Storage unter Verwendung von Trident integriert ist.

Das folgende Diagramm zeigt einen Überblick über die Migration einer VM von VMware zu Red Hat OpenShift Virtualization.

image:rh-os-n-use-case-vm-migration-using-mtv.png["Abbildung, die einen Eingabe-/Ausgabedialog zeigt oder schriftlichen Inhalt darstellt"]



[[prerequisites-for-the-sample-migration]]
=== Voraussetzungen für die Beispielmigration



[[on-vmware]]
=== Auf VMware

* Eine RHEL 9 VM mit RHEL 9.3 und den folgenden Konfigurationen wurde installiert:
+
** CPU: 2, Speicher: 20 GB, Festplatte: 20 GB
** Benutzeranmeldeinformationen: Root-Benutzer und Admin-Benutzer


* Nachdem die VM bereit war, wurde der PostgreSQL Server installiert.
+
** Der PostgreSQL Server wurde gestartet und für den automatischen Start beim Systemstart aktiviert.




[source, console]
----
systemctl start postgresql.service
systemctl enable postgresql.service
----
Diese Befehle stellen sicher, dass der Server nach der Migration in der VM in OpenShift Virtualization gestartet werden kann.

* Es wurden zwei Datenbanken, eine Tabelle und eine Zeile zur Tabelle hinzugefügt. Weitere Informationen zur Installation des PostgreSQL-Servers unter RHEL sowie zum Erstellen von Datenbank- und Tabelleneinträgen enthält link:https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/9/html/configuring_and_using_database_servers/using-postgresql_configuring-and-using-database-servers#configuring-postgresql_using-postgresql["hier,"].



NOTE: Stellen Sie sicher, dass der PostgreSQL-Server gestartet wird und der Dienst so konfiguriert ist, dass er beim Systemstart automatisch startet.



[[on-openshift-cluster]]
=== Auf OpenShift Cluster

Vor der Installation von MTV wurden folgende Installationen durchgeführt:

* OpenShift Cluster 4.17 oder höher
* Multipath auf den Clusterknoten für iSCSI aktiviert (für die Speicherklasse ontap-san).  Multipathing kann einfach aktiviert werden, wenn Sie Trident 25.02 mit dem Node-Prep-Flag installieren.  Sie können sich auf dielink:osv-trident-install.html["Trident -Installationsabschnitt"] für Details.
* Installieren Sie die erforderlichen Backend- und Speicherklassen sowie die Snapshot-Klasse.  Weitere Informationen finden Sie imlink:osv-trident-install.html["Trident -Installationsabschnitt"] für Details.
* link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.19#Virtualization["OpenShift-Virtualisierung"]




[[install-mtv]]
=== Installieren von MTV

Jetzt kann das Migration Toolkit für Virtualisierung (MTV) installiert werden. Die bereitgestellten Anweisungen link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/installing-the-operator["hier,"] bieten Unterstützung bei der Installation.

Die Benutzeroberfläche des Migration Toolkit for Virtualization (MTV) ist in die OpenShift Webkonsole integriert. Es kann auf link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#mtv-ui_mtv["hier,"] verwiesen werden, um die Benutzeroberfläche für verschiedene Aufgaben zu nutzen.



==== Quellanbieter erstellen

Um die RHEL-VM von VMware auf OpenShift Virtualisierung zu migrieren, ist zunächst der Quellanbieter für VMware zu erstellen. Die Anweisungen link:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#adding-providers["hier,"] zur Erstellung des Quellanbieters sind zu beachten.

Zum Erstellen Ihres VMware-Quellanbieters benötigen Sie Folgendes:

* vCenter-URL
* vCenter-Anmeldeinformationen
* vCenter Server-Fingerabdruck
* VDDK-Image in einem Repository


Beispiel für die Erstellung eines Quellanbieters:

image:rh-os-n-use-case-vm-migration-source-provider.png["Abbildung, die einen Eingabe-/Ausgabedialog zeigt oder schriftlichen Inhalt darstellt"]


NOTE: Das Migration Toolkit for Virtualization (MTV) verwendet das VMware Virtual Disk Development Kit (VDDK) SDK, um die Übertragung virtueller Festplatten von VMware vSphere zu beschleunigen.  Daher wird die Erstellung eines VDDK-Images, obwohl optional, dringend empfohlen.  Um diese Funktion zu nutzen, laden Sie das VMware Virtual Disk Development Kit (VDDK) herunter, erstellen ein VDDK-Image und übertragen das VDDK-Image in Ihre Image-Registrierung.

Befolgen Sie die Anweisungenlink:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/prerequisites#creating-vddk-image_mtv["hier,"] um das VDDK-Image zu erstellen und in ein vom OpenShift-Cluster aus zugängliches Register zu übertragen.



==== Zielanbieter erstellen

Der Hostcluster wird automatisch als Zielanbieter hinzugefügt.



==== Migrationsplan erstellen

Befolgen Sie die Anweisungenlink:https://access.redhat.com/documentation/en-us/migration_toolkit_for_virtualization/2.5/html/installing_and_using_the_migration_toolkit_for_virtualization/migrating-vms-web-console#creating-migration-plan_mtv["hier,"] um einen Migrationsplan zu erstellen.

Beim Erstellen eines Plans müssen Sie Folgendes erstellen, sofern dies noch nicht geschehen ist:

* Eine Netzwerkzuordnung zum Zuordnen des Quellnetzwerks zum Zielnetzwerk.
* Eine Speicherzuordnung, um den Quelldatenspeicher der Zielspeicherklasse zuzuordnen. Dafür kann die ontap-san Speicherklasse ausgewählt werden. Nach Erstellung des Migrationsplans sollte der Status des Plans *Bereit* anzeigen und es sollte nun möglich sein, den Plan zu *Starten*.


image:rh-os-n-use-case-vm-migration-mtv-plan-ready.png["Abbildung, die einen Eingabe-/Ausgabedialog zeigt oder schriftlichen Inhalt darstellt"]



[[perform-cold-migration]]
=== Durchführen einer Cold Migration

*Start* auswählen, um eine Abfolge von Schritten auszuführen und die Migration der VM abzuschließen.

image:rh-os-n-use-case-vm-migration-mtv-plan-complete.png["Abbildung, die einen Eingabe-/Ausgabedialog zeigt oder schriftlichen Inhalt darstellt"]

Nach Abschluss aller Schritte können die migrierten VMs angezeigt werden, indem im linken Navigationsmenü unter *Virtualisierung* auf *Virtuelle Maschinen* geklickt wird. Zur Verwaltung der virtuellen Maschinen siehe link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.22/html/virtualization/managing-vms["die OpenShift Virtualisierungsdokumentation"].

Sie können sich in die virtuelle Maschine einloggen und den Inhalt der PostgreSQL-Datenbanken überprüfen. Die Datenbanken, Tabellen und die Einträge in der Tabelle sollten mit denen übereinstimmen, die auf der Quell-VM erstellt wurden.



[[perform-warm-migration]]
=== Warme Migration durchführen

Um eine Warmmigration durchzuführen, nach der Erstellung eines Migrationsplans wie oben gezeigt, sind die Planeinstellungen zu bearbeiten, um den Standardmigrationstyp zu ändern. Auf das Bearbeitungssymbol neben der kalten Migration klicken und die Schaltfläche umschalten, um auf Warmmigration zu setzen. Auf *Speichern* klicken. Jetzt auf *Start* klicken, um die Migration zu starten.


NOTE: Stellen Sie sicher, dass beim Wechsel von Block-Speicher in VMware die Block-Speicherklasse für die OpenShift Virtualisierungs-VM ausgewählt ist. Außerdem sollte volumeMode auf Block und der Zugriffsmodus auf ReadWriteMany (RWX) eingestellt sein, sodass eine Live-Migration der VM zu einem späteren Zeitpunkt möglich ist.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-001.png["1"]

Auf *0 von 1 VMs abgeschlossen* klicken, die VM erweitern, und der Fortschritt der Migration ist sichtbar.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-002.png["2"]

Nach einiger Zeit ist die Festplattenübertragung abgeschlossen, und die Migration wartet auf den Übergang in den Umstellungszustand. Der DataVolume befindet sich im Status „Pausiert“. Zurück im Plan kann die Schaltfläche *Umstellung* ausgewählt werden.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-003.png["3"]

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-004.png["4"]

Im Dialogfeld wird die aktuelle Uhrzeit angezeigt. Die Uhrzeit kann geändert werden, um die Umstellung auf einen späteren Zeitpunkt zu planen, oder mit *Umstellung festlegen* wird die Umstellung sofort durchgeführt.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-005.png["5"]

Nach einigen Sekunden wechselt der DataVolume Status von Paused zu ImportScheduled und dann zu ImportInProgress, wenn die Umstellungsphase beginnt.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-006.png["6"]

Wenn die Umstellungsphase abgeschlossen ist, gelangt das DataVolume in den Zustand Succeeded und die PVC ist gebunden.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-007.png["7"]

Der Migrationsplan wird fortgesetzt, um die ImageConversion-Phase abzuschließen, und schließlich wird die VirtualMachineCreation-Phase abgeschlossen. Die VM erreicht den Status „Running“ in OpenShift Virtualization.

image:rh-os-n-use-case-vm-migration-mtv-plan-warm-008.png["8"]
